## 1. 動機與假說

### 1.1 動機

任何聲稱觀察到「異常推力」「異常重量變化」「重力屏蔽」的實驗，其結論的可信度上限由**測量平台的雜訊本底決定**。在 Boundary Lab 開始任何反重力相關測量之前，我們必須先量化：

- 我們的精密天平在無樣品、無激勵下，72 小時內讀數的隨機與系統漂移
- 環境變數（溫度、濕度、氣壓、振動、市電諧波）與讀數的相關性
- 在哪個頻段、什麼時間尺度上，我們具備分辨「真訊號」的能力

這個實驗本身**不會發現任何反重力現象**。它的價值在於：之後所有實驗的「正面結果」必須顯著超過此處量化的本底，才能被視為真訊號。

### 1.2 工作假說

**假說 H₀**（虛無假說）：天平讀數在 72 小時內的標準差 σ ≤ 0.5 mg，且漂移 < 1 mg/hour，且 PSD 在 0.001–0.5 Hz 段呈典型的 1/f + 白雜訊組合。

**假說 H₁**：天平讀數出現超出規格書宣稱的雜訊（σ > 1 mg 或 漂移 > 2 mg/hour 或頻譜出現未預期共振峰）。

**結果為 H₁ 的處理**：定位雜訊源（環境 / 電源 / 機械 / 天平本身）並重新進行本底量測，直到 H₀ 成立或天平更換。

### 1.3 與 Boundary Lab 整體目標的關係

本實驗的輸出（噪聲本底數值）將成為後續所有實驗報告的引用基準。任何 Phase 1+ 的「異常」聲稱必須對照此處量化的本底。

## 2. 預先註冊的判定條件

**必須在實驗執行前 git tag 凍結，事後不得修改**。

| 指標 | 通過標準（H₀） | 不通過（H₁） |
|------|---------------|--------------|
| 1 小時滾動 σ | ≤ 0.5 mg | > 1.0 mg |
| 24 小時漂移 | ≤ 1 mg | > 2 mg |
| PSD 0.01 Hz 段 | < 10⁻³ mg²/Hz | > 10⁻² mg²/Hz |
| PSD 出現共振峰 | 無 > 5σ 峰 | 任何 > 5σ 峰 |
| 與溫度的線性相關係數 \|r\| | < 0.3 | > 0.5 |
| 與濕度的線性相關係數 \|r\| | < 0.3 | > 0.5 |

「不確定區」（介於兩者之間）：標記為「需延長測量至 168 小時 / 7 天」。

## 3. 已知背景與系統誤差源（不完全列表）

實驗設計時必須考慮以下雜訊源：

| 雜訊源 | 預期影響量級 | 緩解措施 |
|--------|-------------|----------|
| 環境溫度漂移 | 0.1–1 mg/°C（依天平規格） | 雙層保麗龍 + 多點 DS18B20 監測 |
| 氣流（空調、開門） | 0.1–10 mg 瞬時 | Faraday cage 兼防風罩 |
| 大氣浮力變化 | ~0.04 mg/hPa（120 g 樣品） | BME280 記錄、後處理校正 |
| 地震與交通振動 | 變動、頻段 1–10 Hz | 石板 + 內胎隔振平台 |
| 市電 50/60 Hz 諧波 | 視天平 ADC 規格 | RS-232 隔離、UPS、Faraday cage |
| 月球 / 太陽潮汐 | 約 ±0.001 mg | 不緩解，可後處理識別 |
| 天平內部 1/f | 規格書未說明 | 本實驗目的之一即為量化 |
| 軟體採樣抖動 | < 10 μs | NTP 校時 + 採樣時戳記錄 |

## 4. 設備清單與校準

依 `hardware/phase-0-bom.md` 配置。實驗開始前：

- 天平：使用 E2 級 100 g 校準砝碼進行校準，連續執行 5 次以驗證重複性 < 0.2 mg
- 環境感測器：與獨立校準的溫度計 / 氣壓計做交叉比對 ±0.5 °C, ±1 hPa
- 數據鏈路：確認 Raspberry Pi 時鐘 NTP 同步 < 100 ms 偏差

校準資料儲存於 `hardware/calibration-log/00-noise-floor-<YYYYMMDD>.md`。

## 5. 實驗程序

### 5.1 設置（Day 0）

1. 將天平置於石板隔振平台上，平台置於 Faraday cage 內
2. 天平水平校正（內建氣泡）
3. 環境感測器（BME280, DS18B20 ×5）固定在 cage 內各位置
4. 連接 RS-232 → USB → Raspberry Pi
5. 啟動採樣腳本（見 `analysis/00-noise-floor/capture.py`，待寫）
6. 採樣率：天平 1 Hz、環境感測器 0.1 Hz、振動感測器 100 Hz

### 5.2 採集（Day 0–3）

- 連續 72 小時採集，**期間不開啟 cage、不進入工作區**
- 採集前 6 小時設定為「熱穩定期」，分析時剝除
- 任何中斷必須記錄於 `data/raw/00-noise-floor/run-<id>/incidents.md` 並重新開始

### 5.3 分析（Day 3–7）

按 `analysis/00-noise-floor/analyze.py`（待寫）：

1. 載入原始 CSV，產生時序圖
2. 計算 Allan variance（識別 1/f vs 隨機行走主導區段）
3. 計算 Welch PSD（識別共振峰）
4. 與環境變數做相關性分析（Pearson + 滯後 cross-correlation）
5. 與大氣浮力理論值比對

## 7. 盲分析（不適用）

本實驗無「正面 / 負面」二元結論，因此不需要盲分析。但分析者應在計算 Allan variance 與 PSD **之前**先寫好分析腳本並 git commit，避免 p-hacking 式的事後參數調整。

## 8. 停止條件

立即停止實驗的條件：

- 安全：任何 SAFETY.md 中列出的危險徵兆（不太可能，本實驗低風險）
